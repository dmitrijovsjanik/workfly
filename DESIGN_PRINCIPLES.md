# Принципы дизайна Workfly

## Общие принципы UI/UX

### 1. Прогрессивное раскрытие (Progressive Disclosure)
- Показывать только необходимую информацию на каждом шаге
- Скрывать вторичный контент за кнопками "Показать ещё"
- Использовать multi-step формы для сложных процессов (конверсия выше на 14-59%)

### 2. Закон Миллера (Miller's Law)
- Человек комфортно обрабатывает 5-7 элементов одновременно
- Группировать связанные поля вместе
- Разбивать длинные формы на логические секции

### 3. Визуальная иерархия
- Идентификация (фото, имя) всегда сверху
- Основные действия (CTA) выделены визуально
- Деструктивные действия (удаление) внизу страницы

### 4. Минимизация трения
- 2-3 шага максимум для любого действия
- Автозаполнение известных данных
- Inline-валидация в реальном времени

## Принципы форм

### Структура
- **Одна колонка** — лучше для понимания и конверсии
- **Исключение**: короткие связанные поля (город + индекс, мин/макс)
- Порядок от простого к сложному

### Лейблы и подсказки
- Постоянные лейблы (не исчезающие placeholder)
- Примеры формата ("ДД.ММ.ГГГГ")
- Краткий helper-текст для неочевидных полей

### Валидация
- Inline-проверка при переходе между полями
- Чёткие сообщения об ошибках с решением
- Визуальная индикация (зелёная галочка / красная рамка)

### Кнопки
- Основное действие — акцентный цвет, справа
- Вторичные действия — нейтральный стиль
- Всегда показывать состояние загрузки (> 500ms)

## Принципы страницы профиля

### Компоновка (на основе лучших практик)
```
┌─────────────────────────────────────────┐
│  [Аватар] Имя                           │
│  Краткое описание / роль                │
│  [Редактировать]                        │
├─────────────────────────────────────────┤
│  Основная информация                    │
│  (статистика, ключевые метрики)         │
├─────────────────────────────────────────┤
│  Детальные секции (табы или карточки)   │
│  - О себе                               │
│  - Навыки                               │
│  - Портфолио / Заказы                   │
├─────────────────────────────────────────┤
│  Настройки аккаунта (внизу)             │
└─────────────────────────────────────────┘
```

### Режимы редактирования
- **View mode** — чистое отображение данных
- **Edit mode** — форма редактирования (отдельная страница или модал)
- Явная кнопка "Сохранить" с подтверждением

### Статистика и доверие
- Рейтинг, количество выполненных заказов
- Верификация (email, телефон)
- Время последней активности

## Принципы для Workfly

### Профиль исполнителя
Следуем паттернам **Upwork/Fiverr**:
- Фото + имя + краткое био вверху
- Навыки как теги
- Ставка (диапазон)
- Портфолио (если есть)
- Статистика: рейтинг, выполнено заказов

### Панель заказчика
Следуем паттернам **Etsy/Dashboard**:
- Сводка: активных заказов, откликов
- Список заказов с фильтрами
- Быстрое создание нового заказа
- Карточки заказов с основной инфо

### Карточка заказа
- Заголовок + категория
- Бюджет (диапазон)
- Требуемые навыки (теги)
- Статус (бейдж)
- Количество откликов

## Компоненты (Alfa-Bank)

### Основные
- `Typography` — текст с правильной иерархией
- `Button` — view="primary" для CTA, "secondary" для второстепенных
- `Input`, `Textarea` — с label и hint
- `Tag` — для навыков и категорий
- `Card` — для группировки контента
- `Skeleton` — для состояний загрузки

### Обратная связь
- `Toast` — для быстрых уведомлений
- Inline error messages — для ошибок форм
- Loading states — для всех асинхронных действий

## Адаптивность

### Брейкпоинты
- Mobile: < 768px
- Tablet: 768px - 1024px
- Desktop: > 1024px

### Принципы
- Mobile-first подход
- Touch-friendly targets (минимум 44x44px)
- Одна колонка на мобильных
- Sticky header с навигацией
